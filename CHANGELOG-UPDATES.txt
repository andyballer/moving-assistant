Moving Assistant — Product Update Changelog

2026-07-08
- Added first Move Profile fields for city/market, apartment hunt needed, move help, move distance, and building type.
- Added first applicability behavior: apartment-hunt tools/focus hide when not needed, Movers hides for DIY moves, and hidden active tabs redirect to Dashboard.
- Improved Dashboard hierarchy with a primary `Do next` action, secondary `Up next` list, compact status signal cards, and better mobile stacking.
- Extracted Movers rendering, shared tip calculation, and mover/tip event handling into `src/js/movers.js`.
- Added `movers.js` to the app script order, service-worker cache, and JavaScript smoke checks.
- Extracted Box Inventory rendering and event handling into `src/js/boxes.js`, reducing the main app shell and matching the apartment module pattern.
- Added `boxes.js` to the app script order, service-worker cache, and JavaScript smoke checks.
- Added a dependency-free Node test harness with smoke checks for script syntax, cached assets, tab config, state migration, and backup summaries.
- Made JSON backup import safer by rejecting unrelated JSON and showing a move summary before replacing saved data.
- Centralized valid tab IDs in state so navigation and saved-state migration are easier to keep aligned.
- Split apartment hunt/tracker code into its own module so the app is easier to grow.
- Expanded apartment hunting with readiness checklists, listing status tracking, follow-up dates, cashier-check reminders, and search-site guidance.
- Added mover comparison estimates with review links, quote tracking, and all-in totals including suggested tip.
- Improved donation/packing guidance with installed-item reminders, bulky-item donation rules, and checkable donation prompts.
- Hardened local saved-state migration so deployed updates are less likely to lose data on the same browser/site.

Second implementation pass
- Added editable city/market and neighborhood chips to Apartment Hunt so the app is less hardcoded to one NYC move.
- Turned Apartment Hunter guide cards into real checkable action items that count toward progress.
- Added Dashboard quick actions: focus items can now be opened, and many can be marked done directly from the Dashboard.
- Added safer celebration tracking for 50%, 100%, first box, and room-complete moments so confetti does not spam repeatedly.
- Expanded Box Inventory with edit mode, duplicate-label warning, status filters, open-first/fragile counts, and a box summary row.
- Expanded Address & Utilities from simple dates into provider/action/status/account/confirmation/phone/notes cards.
- Added Important Contacts for movers, current building, new building, and emergency backup.
- Expanded Move Day into a richer run sheet: before movers, while loading, final sweep, arrival, and first-night sanity kit.
- Added a mover tipping guide and calculator with crew size, hours, rate, and service quality.
- Added PWA update prompt UI so new service-worker versions can be activated with an in-app update button.
- Renamed the truck asset from “moving truck.webp” to “moving-truck.webp” for safer web paths.
- Removed inline mobile-menu section navigation handlers and replaced them with event listeners.
- Improved first-run behavior so the welcome animation does not cover the onboarding wizard for brand-new users.
